Instructions

User manual for the Label Generator system

Quick Start
  1. Select database in the sidebar (or upload new one in Database tab)
  2. Go to Combined Generator tab
  3. Upload vial image (PNG/JPG)
  4. Upload label template (SVG or AI)
  5. Draw text areas (limits) on the template - see below WHY
  6. Click "Generate All Labels" (Step 1)
  7. Click "Generate Mockups" (Step 2)
  8. Download results from Archive
What This System Does
1. Generates product labels

Takes ONE label template and automatically creates labels for ALL products in your database (e.g., 91 products). Only text changes on each label: product name, ingredients, SKU.

2. Generates realistic mockups

Uses AI (Google Gemini) to place generated labels on vial photos. Result looks like professional product photography.

Final result: From 1 template + 1 vial photo = 91 ready labels + 91 realistic mockups in ~15 minutes
Text Areas (Limits) - Why They're Needed
Why must I draw rectangles?

The system doesn't know where on the label to place text. Your SVG template contains hundreds of elements (backgrounds, shapes, decorations), and the system needs to know EXACTLY where the 2 text fields are that it will replace.

What are text areas (limits)?

Rectangular areas you draw with your mouse on the template. Each rectangle means: "Put product name here" or "Put ingredients here"

Why the system needs them:
  • Rectangle width → How many characters fit per line
  • Rectangle height → How many lines of text fit
  • Rectangle position → Where to center the text
  • Rectangle size → Whether to reduce font size if text is too long
Example

Product 1: "NAD+" (short name) - fits in 1 line
Product 2: "Thymosin Beta 4 Fragment" (long name) - system splits into 2 lines
WITHOUT AREAS: System doesn't know where to break lines and text goes outside label
WITH AREAS: System knows "I have 300px width" and automatically splits text

How to draw areas correctly:
  1. Click "Draw Product Name Area" or "Draw Ingredients Area" button
  2. Click and drag on the template to create a rectangle
  3. Rectangle should EXACTLY cover the area where text should appear
  4. Not too small (text won't fit) and not too large (text will be misaligned)
Common mistakes:
  • Too small rectangle → text truncated or font too small
  • Too large rectangle → text "floats" and looks bad
  • Wrong position → text appears outside the label
Text Alignment (NEW)
What is text alignment?

Text alignment controls whether text appears on the LEFT, CENTER, or RIGHT side of the text area you defined.

Where to find it?

In Template Preview, after drawing text areas, you'll see alignment buttons:

Product Name: [LEFT] [CENTER] [RIGHT]
Ingredients: [LEFT] [CENTER] [RIGHT]
How it works:
  • LEFT - Text starts at the left edge of the text area
  • CENTER - Text is centered in the middle of the text area (default)
  • RIGHT - Text aligns to the right edge of the text area
Example use cases:
  • Product Name: CENTER - for symmetrical, centered design
  • Ingredients: LEFT - for ingredient lists that read from left to right
  • Multi-column layout: LEFT for left column, RIGHT for right column
Important notes:
  • SKU field is NOT controlled - it remains as positioned in the original template
  • Default alignment is LEFT for both Product Name and Ingredients
  • Alignment only affects fields where you drew text areas
  • Change alignment BEFORE generating labels (cannot change after)
Why Must I Convert AI Files?
I have an Adobe Illustrator file (.ai), why can't I use it directly?

.AI files are a proprietary Adobe format. The system CANNOT read .ai files - it only understands SVG (open format, text-based).

What does the converter do?

The converter uses PyMuPDF (PDF library) to:

  1. Open the .ai file (it's actually a PDF format)
  2. Extract all vectors, shapes, colors
  3. Save as SVG (format the system understands)
When is conversion needed?
  • You have a file with .ai extension
  • Designer sent you a file from Adobe Illustrator
  • You don't have access to Illustrator to manually export to SVG
How to convert:
  1. Go to Image Converter tab
  2. Upload .ai file
  3. Click "Convert to SVG"
  4. Download the SVG file
  5. Use this SVG in Combined Generator or Label Generator
Tip:

If you have Adobe Illustrator, it's better to export manually: File → Export → SVG. This gives you more control over export options.

Combined Generator - Step by Step

Generate labels and mockups in one place - most popular tab

Step 1: Upload Vial Image

Upload a clear photo of your vial/bottle. This will be used as the base for mockups. Supported formats: PNG, JPG. Best on white or green background.

Step 2: Upload Label Template

Upload your label design (SVG or AI). If using AI file, convert it first using Image Converter tab. You'll see a preview of the template.

Step 3: Draw Text Areas

Tell the system where to place text:

  1. Click "Draw Product Name Area" button
  2. Click and drag on template to create rectangle around product name
  3. Click "Draw Ingredients Area" button
  4. Draw rectangle around ingredients/dosage area
  5. You can redraw - just click the button again and draw new rectangle
Important: Rectangles must EXACTLY cover the text area. See "Text Areas" section above for details.
Step 4: Generate Labels (STEP 1)

Click green "Generate All Labels" button. System creates a label for each product in your database. Watch progress bar (e.g., "25/91 labels generated"). Processing time: ~2-5 seconds per product. You'll see a gallery of generated labels when done.

Step 5: (Optional) Draw Label Crop Area

If your template has elements that shouldn't be on the mockup (crop marks, "DRAFT" text), click "Draw Label Crop Area" and draw rectangle around just the label. Skip this if entire template should go on mockup.

Step 6: Generate Mockups (STEP 2)

Click orange "Generate Mockups" button. AI (Gemini) places each label on the vial photo. System automatically verifies each mockup (checks if text is correct) and retries if needed (max 3 attempts). Processing time: ~20-30 seconds per mockup (91 products = ~12-15 minutes). Mockups are processed in parallel (4 at a time).

Note: Mockups have transparent background - ready for use on websites or marketing materials.
Step 7: Download Results

Click "Download Labels ZIP" (labels in SVG, PNG, PDF) or "Download Mockups ZIP" (mockups in PNG). Or go to Archive tab to download later.

Done!

You now have complete labels (SVG/PNG/PDF) and realistic mockups (PNG) for all products. Files are named by SKU (e.g., label_YPB.101.png, mockup_YPB.101.png)

Generate only labels (without mockups)

Step 3: Generate Labels

Click "Generate Labels" button. System creates labels for all products in database. Processing: ~2-5 seconds per product.

Step 4: Download

Click "Download ZIP" or go to Archive tab. Labels are in SVG, PNG (300 DPI), and PDF (vector) formats.

  3. Click "Load Labels from Job"

  5. Click "Generate Mockups"
Option 2: Generate from Template
  1. Upload vial image (PNG/JPG)
  2. Upload label template (AI or SVG)
  3. System converts template and generates labels
  4. Then creates mockups automatically
  5. Download ZIP when done
Note:

Mockups use AI (Google Gemini) with automatic retry and verification. Each mockup is checked for text accuracy and regenerated if needed (max 3 attempts).

Image Converter - Step by Step

Convert Adobe Illustrator (.ai) files to SVG

Step 1: Upload AI File

Click "Choose File" and select your .ai file from Adobe Illustrator.

Step 2: Convert

Click "Convert to SVG" button. System uses PyMuPDF to extract vectors and convert to SVG format. Takes ~2-5 seconds.

Step 3: Download SVG

Click "Download SVG" button. Use this SVG file in Combined Generator or Label Generator.

Important:

Converted SVG files may not have editable text (text might be converted to paths). For best results, export SVG manually from Illustrator: File → Export → SVG, with "Convert to Outlines" unchecked.

Database - Step by Step

Manage product databases (CSV files)

Database Structure

Each database is a CSV file with three required columns:

  • Product - product name (e.g., "NAD+", "Thymosin Beta 4")
  • Ingredients - ingredients with dosage (e.g., "NAD+ 100mg")
  • SKU - unique product code (e.g., "YPB.101")
Import New Database
  1. Go to Database tab
  2. Click "+ New Database" button
  3. Select your CSV file
  4. Preview shows detected columns
  5. Map columns (tell system which is Product, Ingredients, SKU)
  6. Enter database name
  7. Click "Import Database"
  8. Database appears in sidebar dropdown
Switch Database

Use dropdown in sidebar to switch between databases. Selected database is used for all label generation.

Edit Products

Click on a product row to edit. Change product name, ingredients, or SKU. Click "Save" to update. Changes are saved to CSV file.

Delete Database

Click trash icon next to database name. Confirm deletion. CSV file is removed from system.

How Text Wrapping Works (Behind the Scenes)
The Algorithm

When you draw text areas, the system uses an intelligent algorithm:

  1. Measures the area you drew (width and height in pixels)
  2. Calculates how many characters fit per line (based on font and width)
  3. Splits text at word boundaries (or "/" for ingredients)
  4. Balances lines to have similar lengths (e.g., 4 ingredients = 2:2 split, not 1:3)
  5. Centers text within the area
  6. If text too long, reduces font size (minimum 18px for ingredients)
Example: Ingredients

Text: "GHRP-2 (5mg) / Tesamorelin 5mg / MGF 500mcg / Ipamorelin 2.5mg"
Algorithm splits:
Line 1: GHRP-2 (5mg) / Tesamorelin 5mg /
Line 2: MGF 500mcg / Ipamorelin 2.5mg
Result: Balanced 2:2 distribution, "/" stays at end of lines

Why this matters:

You don't need to manually adjust text or worry about line breaks. Draw the area, and the algorithm handles everything automatically for all 91+ products.

How Mockup Generation Works (AI-Powered)
The Process
  1. Label is converted to PNG image
  2. Vial photo gets green background (#00FF00) for masking
  3. Both images sent to Google Gemini AI with prompt: "Place this label on the vial realistically"
  4. AI generates photorealistic mockup with perspective, lighting, curvature
  5. System verifies mockup using AI vision (checks SKU, product name, dosage)
  6. If verification fails → retry with error hints (max 3 attempts)
  7. Green background is removed (color-based masking)
  8. Final mockup saved as PNG with transparent background
AI Verification (New Feature)

After generating each mockup, system uses Gemini Vision to read text on the mockup and verify:
✓ SKU matches expected (e.g., "YPB.101")
✓ Product name matches (e.g., "NAD+")
✓ Dosage matches (e.g., "100mg")
If any mismatch → system retries with error hint to AI (e.g., "Previous attempt: SKU incorrect")

Performance

Mockups processed in parallel (4 at a time). For 91 products: ~12-15 minutes total. Each mockup: ~20-30 seconds including verification.

Label Crop Area (Optional)

If template has elements that shouldn't be on mockup (crop marks, "DRAFT" text, guides), draw a crop rectangle to use only part of the label.

Why AI?

Simple image overlay doesn't look realistic. AI understands perspective, lighting, curvature of the vial - result looks like professional product photography.

What's Stored

All generated labels and mockups are automatically saved. Each generation creates a timestamped folder (e.g., labels_20260129_143000).

Actions
  • Download - download entire generation as ZIP
  • View - preview individual files in browser
  • Delete - remove specific generation (frees disk space)
  • Clean Old Files - bulk delete files older than X days
  • Sort - sort by date (newest first), type (labels/mockups), or size

Automatic Cleanup:

System automatically deletes temp files older than 24 hours on startup. Archive files are kept until you manually delete them.

Settings - Gemini API Key
What is the API key for?

API key is used ONLY for mockup generation (AI placing labels on vials). Label generation works WITHOUT API key (uses text algorithms only).

How to Get API Key (Free)
  1. Go to Google AI Studio
  2. Sign in with your Google account
  3. Click "Create API Key" button
  4. Copy the key (starts with "AIza...")
  5. Paste it in Settings tab and click "Save API Key"
API Usage & Cost

Google Gemini API has generous free tier:

  • Free: 1,500 requests/day (enough for ~1,500 mockups)
  • Model used: gemini-2.5-flash-image (fast, good quality)
  • Each mockup = 1 API request (~20-30 seconds)
  • Verification also uses API (separate, cheaper model)
Summary:

Labels = FREE (no API needed). Mockups = Need API key (free tier covers normal usage).

Output Formats - What You Get
Labels (3 Formats)
  • SVG - vector format, infinite scaling, editable in Illustrator/Inkscape
  • PNG - raster format at 300 DPI (print quality), for web/preview
  • PDF - vector format, ready for professional printing
Mockups (1 Format)
  • PNG with transparency - photorealistic, transparent background, ready for websites/marketing
File Naming Convention

label_YPB.101.svg
label_YPB.101.png
label_YPB.101.pdf
mockup_YPB.101.png

Files are named by SKU for easy identification and organization.

Which format to use?
  • PDF - send to printer for physical labels
  • PNG - use on website or for email
  • SVG - further editing in design software
  • Mockup PNG - product photos for marketing/e-commerce
Troubleshooting - Common Issues
❌ Text doesn't fit properly / Text is cut off

Solution: Redraw the text area (limit) larger. The rectangle must match the EXACT boundaries where text should appear.

Common mistake: Drawing area too small. Text area should cover the full width and height available for text.

❌ Mockup generation fails / "API key invalid"

Solution: Go to Settings tab and check API key is configured correctly.

System automatically retries failed mockups 3 times. If still failing after 3 attempts, check API key and internet connection.

❌ Text wrapping looks wrong / Lines too short or too long

Solution: Adjust the WIDTH of your text area. Wider = more characters per line. Narrower = fewer characters per line.

The algorithm uses area width to calculate line breaks. Draw area to match desired text layout.

❌ Progress bar stuck / Page not responding

Solution: Refresh browser (Ctrl+Shift+R or Cmd+Shift+R on Mac). Processing continues in background.

After refresh, check Archive tab - files may have been generated even though progress bar was stuck.

❌ Some products missing in generated labels

Solution: Go to Database tab and check missing products. Every product MUST have all three fields: Product, Ingredients, SKU.

Empty fields cause products to be skipped. Fill in all data and regenerate.

❌ AI file won't convert to SVG / "Conversion failed"

Solution: File may be corrupted or not a valid Adobe Illustrator file. Try opening in Illustrator and manually exporting as SVG.

Alternative: File → Export → SVG in Illustrator with "Convert to Outlines" unchecked.

❌ Mockups look unrealistic / Label not properly placed

Solution: Use higher quality vial photo on plain background. AI works best with clear, well-lit photos on white or green background.

System automatically retries with verification. If quality still poor after 3 attempts, try different vial photo.

❌ Download ZIP is empty / Missing files

Solution: Go to Archive tab and check if generation completed successfully. Look for green checkmark.

If generation failed mid-way, partial files are still saved. Check Archive for what was generated.
